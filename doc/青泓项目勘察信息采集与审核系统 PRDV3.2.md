1. 项目概述
1.1 项目名称
青泓项目勘察信息采集与审核系统

1.2 建设目标
建设一套覆盖 项目管理、点位管理、移动采集、审核闭环、动态模板、导出归档、第三方协作 的勘察业务系统，解决以下问题：

现场采集标准不统一

不同排口类型字段差异大，固定表单无法适配

弱网环境下数据易丢失

定位偏差缺少统一纠偏机制

审核缺少版本留痕与驳回闭环

第三方协作入口不安全、权限不可控

1.3 建设范围
PC 后台管理端：Vue3 + Element Plus

移动采集端：uni-app + uView Plus

第三方协作入口：基于 uni-app / 小程序受限入口

服务端：RESTful API

数据库：MySQL 8.0

文件存储：OSS

地图服务：高德地图 / 微信小程序原生 map

1.4 核心价值
实现标准化采集

实现模板动态配置

实现离线草稿与弱网恢复

实现地图纠偏与位置追溯

实现审核闭环与版本管理

实现导出归档与审计留痕

实现第三方协作安全接入

1.5 项目成功指标
草稿丢失率小于 0.5%

驳回重提版本 100% 可追溯

PDF 报告生成成功率大于 99%

审核记录 100% 保留操作链路

4. 业务流程
4.1 主业务流程
管理员创建项目

配置成员、标段、模板（「优化」设置不同排口类型与模板版本的绑定关系）

导入/新增点位（「优化」点位需录入排口类型）

点位分配给采集员

采集员移动端点击“开始采集”认领任务，点位状态变更为“草稿中”

采集员现场定位、拍照、填表（「优化」表单由点位排口类型自动匹配对应模板版本）、保存草稿

采集员提交审核

系统生成新版本并进入待审核

审核员执行通过或驳回

「优化」驳回后，系统将被驳回版本数据复制为新的草稿版本（version_no+1），供采集员在此基础上修改；原被驳回版本锁定为只读历史

采集员修改新草稿后重新提交，再次进入待审核；若再次驳回，重复第10步

审核通过后，系统自动触发单点位 PDF 生成并归档，同时点位状态更新，纳入可导出范围

4.2 模板流程
创建模板草稿

配置字段、校验规则、联动规则

预览移动端效果

发布模板版本

「优化」在项目/标段下，将发布的模板版本与排口类型建立绑定关系

后续调整模板必须生成新版本，旧版本在已绑定点位中保持可用

4.3 第三方协作流程
创建协作入口

设置授权点位范围、权限、有效期

生成链接/小程序码/临时账号

第三方在授权范围内执行查看/协助填报

系统记录访问和操作日志

到期自动失效或手工撤销

5.2 点位状态（展示层聚合状态）
「优化」明确：点位状态为系统根据其下采集结果的当前状态逻辑生成，不直接物理存储，规则如下：

点位下采集结果最新状态	点位展示状态
无任何采集结果	待采集
最新采集结果为草稿	草稿中
最新采集结果为已提交/待审核	待审核
最新采集结果为已通过	审核通过
最新采集结果为已驳回	驳回待修改
最新采集结果为已归档	已归档
人工作废标记	作废
点位流转规则：

待分配 -> 待采集（分配采集员）

待采集 -> 草稿中（采集员点击“开始采集”）

草稿中 -> 待审核（提交）

待审核 -> 审核通过 / 驳回待修改

驳回待修改 -> 草稿中（采集员进入编辑）-> 待审核（重新提交）

审核通过 -> 已归档（人工或自动归档）

任意有效状态 -> 作废（管理员操作）
