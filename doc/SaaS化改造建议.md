# 青泓项目勘察系统 - SaaS化改造建议

## 一、当前状态

已决定先按单租户版本完成开发，后续再进行SaaS化改造。

## 二、SaaS化改造预留设计

为降低后续SaaS化改造成本，建议在当前开发中预留以下设计：

### 1. 数据库层面
- 所有业务表预留 `tenant_id` 字段（可先设为NULL或默认值0）
- 后续SaaS化时只需启用该字段并配置MyBatis-Plus租户插件

### 2. 实体类层面
- 在BaseEntity中预留 `tenantId` 字段
- 所有实体继承BaseEntity

### 3. 接口层面
- JWT Token中预留 `tenantId` 字段
- 后续SaaS化时只需在拦截器中校验租户

## 三、SaaS化改造预估

### 改造内容
| 改造项 | 工作量 |
|-------|-------|
| 数据库表增加tenant_id | 2-3天 |
| MyBatis-Plus租户插件集成 | 1天 |
| 租户管理模块开发 | 3-5天 |
| 计费计量模块 | 5-7天 |
| 前端租户适配 | 2-3天 |
| 测试与调优 | 3-5天 |
| **合计** | **约16-24天** |

### 推荐架构
```
租户隔离策略：共享数据库 + 共享Schema + tenant_id字段隔离
```
