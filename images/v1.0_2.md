# 亳州入河排污口项目勘察系统 - 全功能设计蓝图 (v1.0)

## 一、 系统架构与视觉标准
本系统采用“PC 管理端 + 移动执行端”的双端协同模式，所有设计严格遵循以下标准：
*   **分辨率**：统一锁定为 **1920x1080**。
*   **导航规范**：PC 端侧边栏固定宽度为 **240px**，确保视觉比例严谨一致。
*   **语言环境**：全界面采用 **纯中文** 交互，确保作业术语对齐（如统一使用“勘察”而非“巡检”）。

---

## 二、 菜单架构方案 (全系统对齐)

### 2.1 管理后台 (PC端)
核心导航采用左侧侧边栏布局，划分为以下功能模块：

1.  **[首页] 概览工作台 (Dashboard)**
    *   展示全域勘察 KPI 指标、任务负载分析、异常告警实时流、作业热力图分布。
2.  **[业务] 地图视图 (Map View)**
    *   全量点位可视化，支持按项目/状态筛选，提供**人工纠偏**（修正 GPS 坐标）功能。
3.  **[项目] 项目管理 (Project Management)**
    *   项目全生命周期管理、负责人指派、数据全局导出。
4.  **[任务] 任务中心 (Task Center)**
    *   勘察任务列表、分派逻辑、执行进度跟踪、逾期预警。
5.  **[团队] 团队管理 (Team Management)**
    *   人员档案、所属单位（环保局/第三方）、绩效考核、权限准入。
6.  **[质量] 审核中心 (Audit Center)**
    *   待审列表、多版本对比、水印照片查验、审核决策（通过/驳回）。
7.  **[配置] 模板引擎 (Template Engine)**
    *   可视化表单配置、字段逻辑联动（如：选中“太阳能”触发“日照条件”）。
8.  **[系统] 系统管理 (System Admin)**
    *   用户角色 (RBAC)、组织架构、消息通知中心、审计日志、字典配置。

### 2.2 移动端勘察系统 (uni-app)
采用底部标签栏导航，确保现场作业高效：

1.  **地图 (Map)**：现场打点、位置自动采集与手动纠偏。
2.  **勘察 (Surveys)**：动态表单填报、离线暂存、硬件相机强制调用。
3.  **任务 (Tasks)**：个人任务清单查看、进度实时上报。
4.  **消息 (Messages)**：审核反馈通知、系统公告、任务变更提醒。
5.  **我的 (Profile)**：个人资料、所属单位、离线地图资源包下载。

---

## 三、 核心补强模块 (上线标准)

### 3.1 认证与消息体系
*   **安全准入**：支持工号/手机号登录，提供注册及角色申请流程，由后台管理员审批。
*   **多端通知**：任务分派、审核结果及系统变更均通过消息中心实时推送，支持 App Push 与 PC 弹窗。

### 3.2 资源与字典管理
*   **素材库**：集中管理带 OSS 水印的现场照片及生成的 PDF 勘察报告。
*   **基础字典**：维护行政区划、河道信息、排口类型等全系统通用的基础数据。

### 3.3 审计与预警
*   **操作审计**：记录点位坐标修改、数据导出等敏感行为。
*   **异常规则**：配置偏移量预警、表单填写异常逻辑自动标红。
